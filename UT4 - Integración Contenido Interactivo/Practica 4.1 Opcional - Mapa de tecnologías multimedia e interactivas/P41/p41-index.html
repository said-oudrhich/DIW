UT4 · Práctica 1 — Mapa de tecnologías multimedia e interactivas

Alumno/a: Said Oudrhich · Curso: 2º DAW · Módulo: DIW

Introducción

En esta práctica se presenta un mapa de tecnologías multimedia e interactivas en la web. Se analiza cómo HTML5, CSS, SVG, Canvas, JavaScript y APIs modernas permiten crear experiencias ricas, accesibles y dinámicas. El objetivo es valorar las aportaciones de cada tecnología y elegir la herramienta adecuada según el contenido y el tipo de interacción deseada.

1. Audio y vídeo en HTML5

Qué es / para qué sirve

HTML5 incorpora las etiquetas <audio> y <video> para reproducir medios directamente en el navegador sin necesitar plug-ins externos. Permiten controlar la reproducción, mostrar controles nativos, añadir subtítulos y adaptarse a diferentes dispositivos.

Casos de uso típicos

Se utilizan en reproductores de música, transmisiones educativas, vídeos tutoriales, galas, anuncios de producto y contenidos multimedia en plataformas de formación. También se usan para podcasts, lecciones online y contenido corporativo.

Ventajas

  • Compatible con navegadores modernos sin plugins.
  • Controles accesibles y opciones de subtítulos/leyendas.
  • Soporte para streaming progresivo y formatos múltiples.
  • Integración sencilla con JavaScript y APIs multimedia.

Limitaciones

  • Compatibilidad de formatos depende de los códecs.
  • Archivos pesados pueden afectar a la carga y al ancho de banda.
  • Experiencias avanzadas requieren JavaScript adicional.

Ejemplos reales (webs conocidas)

Plataformas como YouTube, Spotify y la BBC utilizan HTML5 media como base de sus reproductores. También los cursos de Coursera o edX integran vídeo nativo y pistas de subtítulos.

Conclusión personal

Usaría HTML5 media cuando el contenido principal sea audio o vídeo y se necesite una reproducción sencilla, accesible y compatible con dispositivos móviles. Es ideal para materiales grabados y para dar al usuario control de reproducción sin instalar software adicional.

2. Canvas

Qué es / para qué sirve

Canvas es un elemento HTML5 que crea un área de dibujo en píxeles. Se utiliza para gráficos dinámicos, juegos, simulaciones, visualizaciones interactivas y animaciones que se generan con JavaScript.

Casos de uso típicos

Es adecuado para juegos 2D, efectos visuales, editores de imagen en el navegador, visualizadores de audio y aplicaciones de dibujo. También se usa para renderizar mapas personalizados o gráficos en tiempo real.

Ventajas

  • Control total sobre píxeles y renderizado.
  • Excelente rendimiento en animaciones y gráficos actualizados.
  • Ideal para contenido no estructurado que no necesita DOM.

Limitaciones

  • No es accesible de forma nativa.
  • El contenido no escala tan bien en pantallas de alta resolución sin código adicional.
  • Requiere JavaScript y no es fácilmente indexable por buscadores.

Ejemplos reales (webs conocidas)

Juegos y demos en Kongregate, visualizadores en p5.js y animaciones interactivas en sitios creativos como Codrops emplean Canvas.

Conclusión personal

Elegiría canvas cuando la aplicación exige dibujo pixel-perfect y alta frecuencia de actualización, como en juegos o simuladores. No lo usaría para interfaces basadas en texto o estructura semántica, donde HTML y CSS son más apropiados.

3. SVG

Qué es / para qué sirve

SVG es un formato vectorial basado en XML para gráficos escalables. Se emplea para iconos, ilustraciones, diagramas, logos y animaciones ligeras que mantienen la calidad a cualquier resolución.

Casos de uso típicos

Es perfecto para iconos, gráficos de datos, mapas interactivos, animaciones UI y cualquier diseño que deba escalar sin perder nitidez.

Ventajas

  • Escalable sin pérdida de calidad.
  • Se puede animar y manipular con CSS y JavaScript.
  • Soporta accesibilidad con títulos y descripciones.

Limitaciones

  • Puede ser más lento con muchos elementos complejos.
  • No es la mejor opción para imágenes fotográficas o contenido basado en píxeles.

Ejemplos reales (webs conocidas)

Webs como Mozilla, CodePen y muchos sitios corporativos usan SVG para iconos, logotipos y gráficos interactivos.

Conclusión personal

Usaría SVG para todo lo que deba ser nítido en pantallas grandes o móviles y para ilustraciones interactivas. Es especialmente útil cuando necesito accesibilidad y escalabilidad sin sacrificar calidad.

4. CSS Animations / Transitions

Qué es / para qué sirve

Las transiciones y animaciones de CSS permiten aplicar cambios visuales suaves en propiedades de estilo. Las transiciones funcionan al cambiar de un estado a otro, y las animaciones con @keyframes permiten secuencias más complejas.

Casos de uso típicos

Se usan en efectos hover, menús desplegables, loaders, carruseles, entradas suaves y microinteracciones que hacen la experiencia más fluida sin usar JavaScript.

Ventajas

  • Menor consumo de recursos respecto a animaciones JS.
  • Declarativo y fácil de mantener.
  • Funciona bien en muchos navegadores modernos.

Limitaciones

  • No permite lógica condicional compleja.
  • Limitado para animaciones que dependen de datos dinámicos.

Ejemplos reales (webs conocidas)

Google, Apple y muchas tiendas online usan transiciones y animaciones en botones, tarjetas y menús para una experiencia más natural y atractiva.

Conclusión personal

Emplearía CSS para animaciones de interfaz y efectos visuales sencillos porque son ligeros y tienen buen rendimiento. Para comportamientos más complejos o control preciso, combinaría CSS con JavaScript.

5. JavaScript para interactividad

Qué es / para qué sirve

JavaScript permite detectar eventos, modificar el DOM y responder a acciones del usuario. Es la base de la interactividad moderna en la web.

Casos de uso típicos

Se utiliza para formularios dinámicos, menús interactivos, aplicaciones SPA, filtros de contenido, efectos personalizados y trabaj con APIs del navegador.

Ventajas

  • Permite lógica avanzada y respuestas instantáneas.
  • Accede a APIs de navegador como Web Audio, Geolocation o Storage.
  • Es imprescindible para experiencias personalizadas y SPA.

Limitaciones

  • Depende de que el usuario tenga JavaScript habilitado.
  • Puede aumentar la complejidad y el tiempo de carga.
  • Mal uso puede afectar el rendimiento y la accesibilidad.

Ejemplos reales (webs conocidas)

Gmail, Trello, Spotify Web, Amazon y muchas aplicaciones web dependen de JavaScript para actualizar contenido sin recargar la página y ofrecer interacciones avanzadas.

Conclusión personal

Usaría JavaScript cuando el proyecto necesita interactividad real, lógica o acceso a APIs del navegador. Siempre conviene aplicar progresive enhancement y no sustituir HTML y CSS cuando ellos pueden resolver el diseño.

6. APIs relacionadas

API 1: Web Audio API

Qué hace: Permite crear y procesar audio directamente en el navegador, desde reproductores y efectos hasta visualizadores.

Casos de uso: Mezclar pistas, aplicar filtros, generar sonido dinámico, analizar espectro de audio y crear efectos interactivos.

Ejemplos reales: Demo Web Audio, aplicaciones musicales online, herramientas de creación de sonido.

Conclusión personal: Es una API esencial cuando se necesita control detallado del audio y generar experiencias multimedia inmersivas sin depender de un reproductor externo.

API 2: Intersection Observer API

Qué hace: Detecta cuándo un elemento entra o sale del área visible del navegador sin hacer bucles de comprobación manual.

Casos de uso: Cargar imágenes bajo demanda, activar animaciones al hacer scroll, medir visibilidad de anuncios y mejorar rendimiento de páginas largas.

Ejemplos reales: MDN Intersection Observer, carga perezosa de imágenes en sitios de noticias y portales.

Conclusión personal: Es muy útil para interacciones basadas en scroll y para mejorar la eficiencia al cargar o animar contenido sólo cuando el usuario lo ve.

API 3: MediaDevices / getUserMedia (opcional)

Qué hace: Permite acceder a la cámara y al micrófono del dispositivo con el permiso del usuario.

Casos de uso: Videollamadas, capturas de audio/vídeo, apps de realidad aumentada y experiencias multimedia interactivas.

Ejemplos reales: Aplicaciones de videoconferencia, filtros de cámara, herramientas de grabación y plataformas educativas.

Conclusión personal: Añade un nivel interactivo muy potente para experiencias en tiempo real, pero requiere gestionar permisos y privacidad con cuidado.
